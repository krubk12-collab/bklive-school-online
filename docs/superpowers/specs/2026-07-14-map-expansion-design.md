# โรงเรียนออนไลน์ v7 — ผังจริง + ระบบเวลา/ประตู + ตัวละคร + บาร์ซ้าย + วัด + แก้จอมือถือ

วันที่: 2026-07-14
ไฟล์เกี่ยวข้อง: `index.html`, `data.js`, `build_data.js` (ไฟล์เดียว vanilla JS + canvas ไม่มี framework)

## บริบท

v6.1 (LIVE ปัจจุบัน) มีนักเรียนจริง 544 คน/ครู 43 คน วิ่งบนแผนที่ isometric 100×42 แบ่งซ้าย(ป.)-ขวา(ม.)-ลานเสาธงกลาง งานนี้คือยกเครื่องผังให้ตรงภาพถ่ายดาวเทียมจริงของโรงเรียนชุมชนวัดบางโค + เพิ่มฟีเจอร์ชีวิตประจำวัน (มา-กลับบ้าน, กลางวันกลางคืน, วัด, ผอ./รองผอ., ร้านค้า) + บาร์ค้นหา + แก้บั๊กจอมือถือ

อ้างอิงภาพถ่ายดาวเทียมจริง (ลูกพี่ส่งให้ระหว่างบรีฟ): ประตูหน้า(เหนือ)→สนามหญ้า+โรงยิม+โรงอาหาร(ตะวันตกเฉียงเหนือ)→อาคารประถม→อาคารมัธยม→โดมกิจกรรมเข้าแถว→ประตูหลัง(ใต้ ทางไปวัด)→วัดบางโค+ศาลาวัด→คลองบางใหญ่ ฝั่งตะวันตกมีคลองบางโคขนานยาว

**ไม่ต้องตรงพิกัด GPS เป๊ะ** — ยึดแค่ทิศทาง/ความติดกันของอาคารให้ตรงจริง

## Section 1 — ผังแผนที่ใหม่

จัดแนวเหนือ(บน)-ใต้(ล่าง) แทนซ้าย-ขวาเดิม:

```
                    [ประตูหน้า] ← เข้า-ออกหลัก
   ┌─────────────────────────────────────────────────┐
   │   โรงยิม(ใหม่)  ร้านค้าหน้าประตู(4-5 ร้าน)+7-11    สนามฟุตบอล │
คลอง│   โรงอาหาร(ขยาย)                                    │
บางโค│   ห้องสมุด(ใหม่)   สนามเด็กเล่น(ใหม่)                  │
(ตะวันตก)│  ┌──── อาคารประถม 12 ห้อง (ป.1-ป.6) + ห้องผู้บริหาร ──┐│
   │  └────────────────────────────────────────────────┘│
   │  ┌──── อาคารมัธยม 6 ห้อง + คอม + วิทย์×2 + พักครู ────┐│
   │  │                            โดมกิจกรรมเข้าแถว(เสาธง)─┤│
   │  └────────────────────────────────────────────────┘│
   └─────────────────────────────────────────────────┘
                    [ประตูหลัง] ← ทางไปวัด
                         │
                  วัดบางโค + ศาลาวัด
                         │
                  คลองบางใหญ่ (ข้างวัด)
```

- แผนที่ (GW×GH) ขยายเป็นแนวตั้งยาวขึ้นแทน 100×42 เดิม — ขนาดจริงกะตอนลงมือให้พอดีห้อง/อาคารทั้งหมดโดยคงสัดส่วนห้องเรียน 7×6 เดิม (รองรับนักเรียนจริง ~30 คน/ห้อง)
- ลานเข้าแถว (เสาธง) ย้ายไปอยู่ใต้โดมกิจกรรมแทนลานโล่งเดิม — เสาธงยังอยู่ที่เดิมในโซนนี้
- ห้องผู้บริหาร (ผอ./รองผอ.) อยู่ในอาคารประถม
- ห้องใหม่: โรงยิม, ห้องสมุด, สนามเด็กเล่น, ห้องวิทย์มัธยม 2 (เดิมมี 1), โรงอาหารขยายใหญ่ขึ้น (สัดส่วนกะตอนลงมือให้รองรับคนได้มากกว่าเดิมชัดเจน)
- ร้านค้าหน้าประตูหน้า 4-5 ร้าน + 7-11 (มุมซ้ายบนใกล้ประตูหน้า) — เป็น deco/ป๊อปอัพเบาๆ ไม่ต้องมีตารางสอน
- คลอง (บางโค ฝั่งตะวันตกยาวขนานโรงเรียน, บางใหญ่ ข้างวัด) เป็น deco tile คล้าย pond เดิม ไม่ใช่ room เดินผ่านไม่ได้
- วัดบางโค + ศาลาวัด อยู่นอกรั้วโรงเรียน ใต้ประตูหลัง — เป็นอาคาร deco (ยังไม่ผูกตารางเวรวันพระ)

## Section 2 — เวลา + ประตู + กลางวันกลางคืน

- **ขยายแกนเวลา**: จาก 08:00-16:30 (TMIN/TMAX เดิม) เป็น **06:00-19:00**
- **มาโรงเรียนทยอย**: แต่ละคนสุ่มเวลามาถึงในช่วง 06:00-07:30 (เล่นอิสระในสนาม/สนามเด็กเล่นหลังเข้าประตู) จากนั้น 07:30-07:50 ทยอยเดินเข้าแถวที่โดม ครบจริง ~07:50 เข้าแถวจริง 08:00 ตามเดิม — ก่อนถึงเวลามาของตัวเอง ตัวละครจะยังไม่ปรากฏบนแผนที่ (state ใหม่ "ยังไม่มา")
- **2 ประตู**: นักเรียนส่วนใหญ่ใช้ประตูหน้า, กลุ่มเล็ก (สุ่ม ~15-20% แบบ hash เหมือน sched อื่นๆ ในโค้ด) ใช้ประตูหลัง (ฝั่งวัด) — ครูใช้ประตูหน้าทั้งหมด
- **เลิกเรียนทยอย**: ป. สุ่มออกช่วง ~15:30-16:00, ม. สุ่มออกช่วง ~16:00-16:30 — แต่ละคนสุ่มนาทีออกของตัวเอง เดินไปประตูที่กำหนดแล้วหายไป (ไม่ใช่ออกพร้อมกันทีเดียว) แทนพฤติกรรมเดิมที่หลังเลิกเรียนจะไปวิ่งเล่นเฉยๆไม่มีที่สิ้นสุด
- **ครูทยอยกลับ**: หลัง 16:30 ทยอยออกจนแผนที่ว่างก่อน ~18:00
- **18:00 เป็นต้นไป**: แผนที่ว่างเปล่าจริง — ไม่ทำระบบครูเวรกลางคืนตอนนี้ (ลูกพี่จะเพิ่มข้อมูลให้ทีหลัง)
- **แสงเปลี่ยนตามเวลา**: ขยาย tint overlay ที่มีอยู่แล้ว (`frame()` มี TINT ตาม tm อยู่แล้ว) ให้ไล่โทนครบช่วงใหม่: ฟ้าเช้ามืด 06:00 → แดดจัดเที่ยง → ส้มพลบค่ำ 18:00 → มืดน้ำเงินเข้ม 19:00

## Section 3 — ตัวละคร + พฤติกรรม + ร้านค้า

- **หมาแมวไก่ (Critter)**: แก้บั๊กรากเหตุ — วาด emoji ทับด้วย `fillStyle` เงาที่ค้างจากก่อนหน้า (ดำโปร่ง 20%) ทำให้จางแทบมองไม่เห็น แก้จุดเดียว (reset fillStyle ก่อน fillText) สีจะเข้มปกติทันที
  - เพิ่มหันทิศทางอย่างน้อย 4 ทิศ (ตามทิศที่กำลังเดิน คล้าย `dir` ของ Char) + ท่วงท่าเดิน/วิ่งสมจริงขึ้น (โยก/ก้มตามจังหวะก้าว แทนแค่ลอยขึ้นลง)
  - เสียงร้อง: ใช้บับเบิ้ลข้อความแบบ emote เดิมที่มีอยู่แล้ว (ไม่มีเสียงจริงในระบบ ยกเว้นกริ่ง) — แมว "เหมียว~", หมา "โฮ่ง!", ไก่ตอนเช้า (ก่อน ~07:00) ขัน "เอ้กอีเอ้กเอ้ก" เวลาอื่น "กระต๊าก"
- **ผอ./รองผอ.**: ตัวละครใหม่ 2 ตัว (ไม่อยู่ใน 43 ครูจาก data.js เพราะไม่มีคาบสอน) ประจำห้องผู้บริหาร เดินสุ่มตรวจตราไปตามอาคาร/ห้องต่างๆ ทั้งวัน (พฤติกรรมสุ่มคล้าย Critter free-roam แต่เข้าห้องเรียนได้) — เชื่อมตารางนิเทศจริงเป็นงานถัดไปเมื่อมีลิงก์ (ไม่ทำตอนนี้)
- **นักเรียนไม่เดินพล่านตอนเรียน**: `Char.update()` ตอนนี้ให้ wander สุ่มตลอดไม่ว่ากำลังมีคาบเรียนหรือไม่ — แก้ให้เช็คว่ามีคาบเรียนอยู่ (ห้องมีครูสอน subj) ถ้าใช่ให้หยุด wander นั่งนิ่งที่โต๊ะ ถ้าเป็นพัก/ว่าง/เลิกเรียนถึง wander ได้ตามเดิม
- **ครูยืนสอนหน้าห้อง + ท่าสอน**: ตอนกำลังสอน (มี `subj`) หยุด wander ไปยืนที่จุดหน้ากระดานดำ (ใกล้ผนังกระดานที่มีอยู่แล้วใน `drawBack`) + สลับท่าทางประกอบ (เช่น ยกแขนชี้กระดาน) เป็นจังหวะ
- **ร้านค้าหน้าประตู**: 4-5 แผงลอย deco ใกล้ประตูหน้า แสดงเฉพาะช่วงมา (06:00-08:00) และเลิกเรียน (15:00-18:00) ตามพฤติกรรมจริง
- **7-11**: อาคารใหม่มุมซ้ายบนใกล้ประตูหน้า (deco/popup เบาๆ)
- **ซีนพิเศษครูณัฐธเนศ (T013)**: ช่วงพักเที่ยงทุกวัน เดินออกจากอาคารไปที่ 7-11 พร้อมบับเบิ้ลข้อความ (เช่น "มาตรวจความเรียบร้อยที่เซเว่นครับ") แล้วกลับที่ทำงานตามปกติ — เคสพิเศษเฉพาะครูคนนี้ (match ด้วยชื่อ)

## Section 4 — บาร์ซ้าย + ค้นหา

- แถบด้านซ้ายแบบถาวร พับ/กางได้ (ประหยัดพื้นที่จอมือถือ) 2 ส่วน:
  - **รายชื่อห้อง/อาคาร** — คลิก → กล้องเลื่อนไปห้องนั้น + เปิดป๊อปอัพเดิม (`showRoom`)
  - **รายชื่อคน** แบ่งกลุ่มตามห้องเรียน 18 ห้อง พับ/ขยายได้ต่อห้อง (กันรายชื่อยาวเกิน 544 แถวรวด)
- **ช่องค้นหา** บนแถบ พิมพ์ชื่อครู/นักเรียน → กรองสดทันที คลิกผลลัพธ์ → กล้องเลื่อนไปหาตัว + ไฮไลต์ (ใช้ `hlId` เดิม)
- ฟังก์ชันใหม่ "เลื่อนกล้องไปจุดที่กำหนด" (ตอนนี้มีแค่ `fitCam()` ที่ครอบทั้งแผนที่ ยังไม่มีเลื่อนไปจุดเดียวแบบคงระดับซูมเดิม)

## Section 5 — แก้จอมือถือภาพแตก/เบลอ

สาเหตุ 2 อย่างรวมกัน:
1. `resize()` ตั้ง `cv.width/height` เท่าพิกเซล CSS ตรงๆ ไม่คูณ `devicePixelRatio` — จอมือถือ DPR 2-3x เลยเรนเดอร์ที่ความละเอียดต่ำกว่าจริง ดูฟุ้ง
2. `fitCam()` พยายามยัดทั้งแผนที่ (ใหญ่ขึ้นอีกจากผังใหม่) ใส่จอแคบ ซูมเล็กจิ๋วจนรายละเอียด/ตัวอักษรเป็นจุดพร่า

แก้:
- ปรับ `resize()`/`ctx` ให้คูณ `devicePixelRatio` ก่อนวาด (คมชัดขึ้นทันที ไม่กระทบ logic เดิม)
- จอแคบ (มือถือ) เปิดมาซูมเข้าเฉพาะโซนหลัก (เช่น จุดเข้าแถว/อาคารประถม) แทนยัดทั้งผืนแบบ `fitCam()` เดิม แล้วให้ผู้ใช้ลากซูมดูส่วนอื่นเอง
